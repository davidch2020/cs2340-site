Study Buddy

Introduction

Studying is hard. However, poor organization makes it harder. The saying goes, "you can't improve at something you don't measure". Yet, many open a textbook and start reading aimlessly. Study Buddy is a study tracking app for students. The purpose of the app is to create and track study goals. We aim to create an app with ease-of-use and organized learning at its core. Users can set daily or weekly goals, log study sessions, track their progress, and study with other group members. We developed Study Buddy in Android Studio using Java.

Design & Architecture

Study Buddy uses the Model-View-ViewModel (MVVM) architecture. MVVM is commonly used in Android projects. Model holds the data and business logic of the application. View represents the app's UI including all the XML files and Activities. ViewModel acts as a bridge between Model and View. The ViewModel handles user interaction from the View and passes data to the Model. It is also solely responsible for interacting with Firebase. Throughout the semester, we learned about different diagrams used to layout our app. We implemented these diagrams.

A sequence diagram models a single scenario. It shows interactions between a user and system objects using messages. In this example, I created a sequence diagram of the log study session scenario. The user would send their information to the ViewModel. The ViewModel calls on the Model for validation logic, checking for empty boxes or improper dates. After receiving confirmation all inputs are correct, the information is stored in Firebase. Fragment boxes are used to indicate diverging paths. For example, the 'alt' box has two paths: one path if a condition is met and another if it isn't. I learned to use the sequence diagram as a starting point rather than a final piece. Looking back at the sequence diagram, we made changes to the class name, messages, and interactions as we developed our app. Design should be used to focus the app's scope but changes should be made as you implement.

User Interface

Startup Screen

The first screen the user sees. It allows the user to continue to the login/registration page ("Start") or safely exit the app ("Quit").

Login and Registration Screen

User can input their account username and password. If they don't have one, they can create a new account. The user proceeds to type in their full name, email address, and password. All their data is securely stored in Firebase Authentication.

Dashboard: Streaks and Calendar

The Dashboard is the first screen the user sees after logging into the app. There is a progress bar tracking daily and weekly goal completion. Underneath is two streak trackers. The daily streak counts the number of days in a row a daily goal was completed. The weekly streak counts the number of weeks in a row a weekly goal was completed.

Goal List and Creation

The user can see all the goals they created. Every goal is marked as completed (✅) or not completed (❌) They can create a new goal by clicking the '+' button in the top-right corner. The goal creation screen prompts them for the goal title, subject, duration, goal type (daily vs. weekly), and start date.

Study Log: Complete Your Goals

The study log allows users to make progress towards their goal. Every time a user studies, they can jot down how long they studied and which goal. They can optionally add any notes.

Progress Screen: Track Your Goals

The progress screen tracks users study time from the past five days in minutes. Each day is split between time spent in group study sessions versus personal study sessions. Using the feature, users can track how much time they spent a day studying and if they spent more time studying by themselves or in a group.

Group Study: Never Work Alone

From the group study screen, the user can view a list of their upcoming group study sessions. The screen is split between accepted sessions and pending invites. A yellow circle appears next to invites. After accepting an invite, the session appears under 'Sessions' with a green circle. Clicking on the '+' button in the top-right corner brings the user to the session creation screen. A user inputs the necessary information (title, topic, date, etc.). Clicking on a session under 'Sessions' brings the user to a 'Session Details' page. It contains general information about the session (topic, date, etc.). If the user created the session, they can invite members. They can also cancel the session.

Conclusions and Reflections/Learning

This summer's CS 2340 project was one of the most enjoyable learning experiences I have had. Unlike past college classes I have taken, CS 2340 is a project-based class. Even though we have instructor-guided lectures, a large portion of our time is spent within our groups. This brought with it new challenges.

  • How would we split the work?
  • What do we do if someone doesn't do their work?
  • How can we set intra-team deadlines?

We learned to overcome these challenges. We split the work evenly but assigned responsibilities based on who was most likely to complete their work on time. If someone did not do their work, we were honest on our CATME surveys. We discussed with that person one-on-one. We set intra-team deadlines using Trello. At the beginning we noticed these intra-team deadlines were being missed because they were not enforced. Starting from Sprint 3, we agreed to close our GitHub repo's 'main' branch 24 hours before the deadline. This forced us to finish our work a day before the deadline, allowing more time to test. From my experience working on this project, I learned discipline, organization, and detail-orientedness matter. I had to be disciplined with deadlines, following MVVM, and routinely checking GitHub for pull requests. I had to be organized and detail-oriented because poor code organization could snowball as coupling increased. I contributed to the design, user interface, setting up Firebase, and organizing tasks for team members. It was a lot of work but rewarding.
